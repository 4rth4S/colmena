The Problem

Three gaps Claude Code does not close natively

~100
permission prompts per session

Every Read, Grep, and git status asks for approval. You approve 70 without thinking. Then you miss the one that matters.

Binary trust does not scale: trust everything or approve everything. No middle ground.

?

No quality signal

One agent produces code, another produces more code. Nobody reviews anything. You trust the output because you are tired, not because it was validated.

No cross-agent review. No metrics. No quality signal.

/dev/null

No institutional memory

Findings from one session disappear. The next audit starts from zero. Knowledge does not accumulate between missions.

Every session is an island. No playbook, no repeatable workflow, no history.

Estimated: ~70% of prompts in a multi-agent CC session are permission noise. The rest is real work.

Colmena solves these three gaps with a deterministic firewall, a centralized auditor review protocol, and an ELO-calibrated trust system — all local, all auditable, all filesystem-based.

The Approach

Deterministic governance + ELO-calibrated trust + multi-agent accountability

Domain-Bounded Agents

Each agent gets tool access scoped to its role. The pentester runs nmap but cannot push code. The developer edits files but cannot access secrets. Boundaries are enforced by YAML rules, not agent goodwill.

trust_circle restricted blocked

Deterministic Firewall

Every tool call is evaluated against compiled regex rules. Same input always produces the same decision. No LLM calls, no heuristics, no surprise approvals. Under 15ms per decision.

<15ms zero LLM

ELO Trust Calibration

Trust is earned through audit history, not declared in config. Agents with consistently high review scores earn Elevated status. Agents with poor scores get Restricted or Probation. Human YAML overrides always win.

5 trust tiers temporal decay

Manifest-driven

Declarative YAML describes squad composition, scope boundaries, budget, and acceptance criteria. Reproducible across sessions.

Centralized Auditor

One auditor per mission evaluates all artifacts. QPC framework (Quality + Precision + Comprehensiveness). SubagentStop enforces review before exit.

Tamper-Evident Audit

Every decision written to audit.log with matching rule ID. Append-only JSONL. Any auditor can replay and explain why a call was allowed, asked, or blocked.

Architecture

Trust Firewall

YAML rules, compiled regex, chain-aware Bash, under 15ms

Every tool call passes through the firewall before execution. Rules evaluated in strict precedence order.

# Precedence (first match wins): blocked > delegations > agent_overrides > runtime_agent_overrides > ELO > restricted > chain_aware > chain_guard > mission_revoked > trust_circle > defaults # Chain-aware Bash eval (M7.10) mkdir dir && cd dir && git clone ... -> split by &&/||/;/| (quote-aware) -> each piece re-evaluated -> any block -> whole command blocked -> any ask -> whole command asks -> subshells $(...) -> chain_guard (ask) -> bare KEY=value -> auto-approve

Safety guarantee

Any hook error returns "ask", never "deny". The human always holds the keys on edge cases.

Auto-elevate (M7.15)

Say Y twice to the same command skeleton within the window. The third call is auto-approved with a session-scoped delegation. No config change needed.

Runtime-agent-overrides

When a manifest spawns a mission, per-agent scope and bash patterns are written to runtime-agent-overrides.json. Cleared on deactivate or abort.

Review cycle in detail

# 6-step cycle enforced by SubagentStop: Step 1: Agent works Scoped reads, writes, bash within role limits. All tool calls pass through firewall. Auditor not needed here -- trust rules handle it. Step 2: review_submit Agent calls MCP with artifact + SHA256 hash. Available_roles filters eligible reviewers. -> Reviewer assigned (random, non-author, non-reciprocal) Step 3: SubagentStop check Worker tries to stop: review_submit done? yes -> approve If reviewer: pending evaluations? -> BLOCK until evaluate Step 4: review_evaluate Reviewer calls MCP with QPC scores + findings. Trust gate: avg >= 7.0 && no critical -> auto-complete Otherwise -> alerts.json (human review) Step 5: ELO update Author rating adjusts by score delta. Reviewer earns +5 per finding. Append-only JSONL, never mutated. Step 6: Findings stored Each finding written to findings store. Queryable by role, category, severity, mission.

Mission Gate

Auto-activates when role-scoped delegations exist. Agent tool calls without mission marker ask. Opt-out with enforce_missions: false in YAML.

Security invariants

1 All paths must be absolute, no ".."
2 Hard-blocked prefixes: /etc, /root, /proc, /sys, /boot, /dev
3 Hard-blocked suffixes: .ssh, .aws, .kube, .gnupg
4 Bash sentinels blocked in extra_allow (rm -rf /, dd, mkfs, chmod 777)
5 Forbidden fields (role_type, etc.) rejected at parse time
ELO Calibration

Append-only log, temporal decay, 4 trust tiers, earned not declared

ELO turns review scores into a trust signal. The cycle is closed: agents produce work, auditor reviews, ELO updates, trust recalibrates. No manual override needed.

ELO engine

# Author delta (by average score): score >= 8 --> +(score - 7) * 3 (reward) score 5-7 --> 0 (neutral) score < 5 --> -(6 - score) * 4 (penalty) # Finding penalties on author: critical --> -10 | high --> -5 # Reviewer reward per finding: any finding --> +5

Trust tiers (min 3 reviews)

Elevated (ELO >= 1600) — Auto-approve role tools_allowed

Standard (ELO 1300-1599) — Default rules

Restricted (ELO 1100-1299) — Everything asks

Probation (ELO < 1100) — Bash + WebFetch blocked

Uncalibrated (reviews < 3) — Default rules, warming up

Safety properties

Blocked always wins. YAML beats ELO. Elevated Bash guard (needs explicit patterns). Calibrate reset kills all overrides. Trust gate floor at 5.0.
